# Testing Strategy

LiteBI uses a comprehensive testing approach to ensure stability, especially given the complex state management and client-side database engine.

## Tools
- **Vitest:** Primary test runner (fast, compatible with Vite).
- **Vue Test Utils:** For mounting and interacting with Vue components in isolation.
- **jsdom:** Provides a simulated DOM environment for Vitest.
- **@vitest/coverage-v8:** For generating code coverage reports.

## Testing Layers

### 1. Unit Tests (Logic & Stores)
Testing isolated pure functions and Pinia stores without mounting UI.

- **Target:** `src/stores/`, `src/utils/`
- **Focus:** State transitions in Pinia, mathematical/statistical utility functions, and text parsing logic.
- **Mocking:** We heavily mock `localForage` and `DuckDB` at this layer to ensure tests run instantly.

### 2. Component Tests (UI)
Testing individual Vue components to ensure they render correctly and emit the right events.

- **Target:** `src/components/base/`
- **Focus:** Props validation, event emission, conditional rendering.
- **Example:** Ensure `BaseButton.vue` triggers a click event only when not `disabled`.

### 3. Integration Tests (Engine & Data)
Testing the interaction between Vue/Pinia and the DuckDB WASM engine.

- **Target:** `src/engine/`
- **Focus:** Verifying that a specific SQL query generated by the UI actually returns the correct JSON result set from a mocked DuckDB instance.

## Running Tests

Execute all tests:
```bash
npm run test:unit
```

Run tests with hot-reload (Watch mode):
```bash
npm run test
```

Generate a coverage report:
```bash
npm run test:coverage
```
The report will be generated in the `coverage/` directory and can be viewed in a browser.

## Guidelines for Writing Tests

- Every new Base UI component must have a corresponding `.spec.js` file.
- When fixing a bug, write a failing test first that reproduces the bug, then apply the fix to make it pass (TDD approach).
- Do not write UI tests that rely heavily on `setTimeout`. Use `nextTick()` or `flushPromises()` provided by Vue Test Utils.
